# Track 04 — Arquitetura de Software

Projete sistemas que sejam fáceis de manter, testar e escalar. Abrange princípios, padrões e estilos arquiteturais.

**Tempo estimado:** 3–4 semanas

---

## Tópicos

1. [Princípios SOLID](solid-principles.md) — SRP, OCP, LSP, ISP, DIP com exemplos em TypeScript
2. [Clean Architecture](clean-architecture.md) — camadas, regra de dependência, ports e adapters
3. [Arquitetura Hexagonal](hexagonal-architecture.md) — ports, adapters, primário vs secundário
4. [Domain-Driven Design](domain-driven-design.md) — linguagem ubíqua, bounded contexts, aggregates
5. [Padrões de Projeto: Criacionais](design-patterns-creational.md) — Singleton, Factory, Builder, Prototype
6. [Padrões de Projeto: Estruturais](design-patterns-structural.md) — Adapter, Decorator, Facade, Proxy
7. [Padrões de Projeto: Comportamentais](design-patterns-behavioral.md) — Strategy, Observer, Command, State
8. [Microservices vs Monólito](microservices-vs-monolith.md) — trade-offs, strangler fig, fronteiras de serviço
9. [Versionamento de API e Contratos](api-versioning-contracts.md) — breaking changes, OpenAPI, contract testing
